C#에서 Thread와 Parallel.ForEach 안전하게 중단하는 방법

2017년 1월 9일 · 약 6분
Jeongyong Park
쌍팔년생 개발자

GUI 환경에서 버튼 클릭과 같은 이벤트로 시간이 오래 걸리는 작업을 실행할 때, UI 스레드와 분리하여 개발하는 것은 C#뿐만 아니라 모든 GUI 프레임워크에서 기본적인 패턴입니다.

시간이 오래 걸리는 작업을 더욱 빠르게 처리하고 싶다면 병렬 처리가 가장 효과적인 방법 중 하나입니다. 하지만 "작업을 도중에 안전하게 중단하고 싶다"는 요구사항이 생기면서 복잡한 문제가 발생합니다.

TL;DR: Thread.Abort()는 위험하므로 사용하지 마세요. 대신 CancellationToken과 Task 기반 비동기 패턴을 사용하여 안전하게 병렬 작업을 중단할 수 있습니다.

이 글에서는 Thread 내부의 Parallel.ForEach 작업을 안전하게 중단하는 방법과 현대적인 비동기 패턴을 소개합니다.

Thread.Abort() 사용 금지

중요: 이 글의 원본 코드는 2017년에 작성되었으며, Thread.Abort() 사용을 포함하고 있습니다.

이 방법은 현재 권장되지 않으며, .NET Core/.NET 5+ 에서는 지원되지 않습니다.

이 글에서는 CancellationToken과 Task 기반 비동기 패턴을 사용한 안전한 방법을 제시합니다.

Thread.Abort()를 사용하면 안 되는 이유:​

  • 데이터 손상: 작업 중인 데이터가 불완전한 상태로 남을 수 있음
  • 리소스 누수: 파일 핸들, 네트워크 연결 등이 제대로 해제되지 않을 수 있음
  • 예측 불가능한 동작: 언제 어디서 중단될지 예측할 수 없음
  • .NET Core 미지원: .NET Core/.NET 5+ 에서는 PlatformNotSupportedException 발생

🎯 핵심 개념​

  • CancellationToken: .NET에서 권장하는 안전한 작업 중단 메커니즘
  • Task: Thread보다 현대적이고 안전한 비동기 작업 단위
  • Parallel.ForEach: CPU 집약적 작업의 병렬 처리
  • ConfigureAwait: UI 스레드 데드락 방지

📝 레거시 코드 (참고용 - 사용 금지)​

다음은 원본 글의 코드입니다. 실제 프로젝트에서는 사용하지 마세요:

레거시 코드 보기 (Thread.Abort 사용 - 권장하지 않음)
// ❌ 잘못된 방법 - Thread.Abort() 사용
public class LegacyExample
{
    public void BadExample()
    {
        var canceler = new RulyCanceler();
        var worker = new TestFilter(10000000);

        Thread t = new Thread(() =>
        {
            try
            {
                worker.execute(canceler);
            }
            catch(OperationCanceledException)
            {
                Console.WriteLine("Canceled!");
            }
        });
        
        t.Start();
        Console.ReadKey();
        
        // ❌ 위험한 방법
        canceler.Cancel();
        t.Abort();  // .NET Core에서 지원되지 않음
        t.Join();
    }
}

public class RulyCanceler
{
    private readonly object _cancelLocker = new object();
    private bool _cancelRequest;
    
    public bool IsCancellationRequested
    {
        get { lock (_cancelLocker) return _cancelRequest; }
    }

    public void Cancel() { lock (_cancelLocker) _cancelRequest = true; }

    public void ThrowIfCancellationRequested()
    {
        if (IsCancellationRequested) throw new OperationCanceledException();
    }
}

✅ 현대적이고 안전한 방법​

CancellationToken을 사용하여 안전하고 예측 가능한 작업 중단을 구현하세요.

1. CancellationToken을 사용한 안전한 중단​

using System;
using System.Collections.Concurrent;
using System.Threading;
using System.Threading.Tasks;

namespace ModernThreadingExample
{
    public class SafeParallelProcessor
    {
        public async Task ProcessDataAsync(long itemCount, CancellationToken cancellationToken = default)
        {
            const int batchSize = 1000;
            var partitioner = Partitioner.Create(0, itemCount, batchSize);
            
            // ParallelOptions를 사용하여 CancellationToken 전달
            var parallelOptions = new ParallelOptions
            {
                CancellationToken = cancellationToken,
                MaxDegreeOfParallelism = Environment.ProcessorCount
            };

            try
            {
                await Task.Run(() =>
                {
                    Parallel.ForEach(partitioner, parallelOptions, (range, state) =>
                    {
                        for (long i = range.Item1; i < range.Item2; i++)
                        {
                            // 주기적으로 취소 요청 확인
                            cancellationToken.ThrowIfCancellationRequested();
                            
                            // 실제 작업 수행
                            ProcessSingleItem(i);
                            
                            // 선택사항: 진행률 보고
                            ReportProgress(i, itemCount);
                        }
                    });
                }, cancellationToken);
            }
            catch (OperationCanceledException)
            {
                Console.WriteLine("작업이 취소되었습니다.");
                throw; // 호출자에게 취소 상태 전달
            }
        }

        private void ProcessSingleItem(long index)
        {
            // 실제 작업 로직
            Console.WriteLine($"Processing item {index}");
            
            // 시뮬레이션을 위한 지연
            Thread.Sleep(10);
        }
        
        private void ReportProgress(long current, long total)
        {
            if (current % 1000 == 0)
            {
                var percentage = (double)current / total * 100;
                Console.WriteLine($"진행률: {percentage:F1}%");
            }
        }
    }
}

2. WPF/WinForms에서 사용하는 방법​

다음은 WPF 애플리케이션에서 안전한 병렬 처리를 구현하는 방법입니다:

public partial class MainWindow : Window
{
    private CancellationTokenSource _cancellationTokenSource;
    
    private async void StartButton_Click(object sender, RoutedEventArgs e)
    {
        try
        {
            StartButton.IsEnabled = false;
            CancelButton.IsEnabled = true;
            
            _cancellationTokenSource = new CancellationTokenSource();
            var processor = new SafeParallelProcessor();
            
            // UI 스레드를 차단하지 않고 백그라운드에서 실행
            await processor.ProcessDataAsync(1000000, _cancellationTokenSource.Token)
                          .ConfigureAwait(false);
                          
            MessageBox.Show("작업이 완료되었습니다!");
        }
        catch (OperationCanceledException)
        {
            MessageBox.Show("작업이 취소되었습니다.");
        }
        catch (Exception ex)
        {
            MessageBox.Show($"오류 발생: {ex.Message}");
        }
        finally
        {
            StartButton.IsEnabled = true;
            CancelButton.IsEnabled = false;
            _cancellationTokenSource?.Dispose();
        }
    }
    
    private void CancelButton_Click(object sender, RoutedEventArgs e)
    {
        _cancellationTokenSource?.Cancel();
    }
}

3. 진행률 보고가 포함된 고급 버전​

사용자 경험을 향상시키기 위해 진행률 보고 기능을 추가할 수 있습니다.

public class AdvancedParallelProcessor
{
    public async Task ProcessDataWithProgressAsync(
        long itemCount, 
        IProgress<ProgressReport> progress = null,
        CancellationToken cancellationToken = default)
    {
        const int batchSize = 1000;
        var partitioner = Partitioner.Create(0, itemCount, batchSize);
        var processedCount = 0L;
        
        var parallelOptions = new ParallelOptions
        {
            CancellationToken = cancellationToken,
            MaxDegreeOfParallelism = Environment.ProcessorCount
        };

        try
        {
            await Task.Run(() =>
            {
                Parallel.ForEach(partitioner, parallelOptions, (range, state) =>
                {
                    for (long i = range.Item1; i < range.Item2; i++)
                    {
                        cancellationToken.ThrowIfCancellationRequested();
                        
                        ProcessSingleItem(i);
                        
                        // 스레드 안전한 카운터 증가
                        var current = Interlocked.Increment(ref processedCount);
                        
                        // 진행률 보고 (너무 자주 호출하지 않도록 조절)
                        if (current % 100 == 0 || current == itemCount)
                        {
                            var percentage = (double)current / itemCount * 100;
                            progress?.Report(new ProgressReport
                            {
                                Current = current,
                                Total = itemCount,
                                Percentage = percentage,
                                Message = $"처리 중... ({current:N0}/{itemCount:N0})"
                            });
                        }
                    }
                });
            }, cancellationToken);
        }
        catch (OperationCanceledException)
        {
            progress?.Report(new ProgressReport
            {
                Current = processedCount,
                Total = itemCount,
                Percentage = (double)processedCount / itemCount * 100,
                Message = "작업이 취소되었습니다."
            });
            throw;
        }
    }
    
    private void ProcessSingleItem(long index)
    {
        // 실제 작업 로직
        Thread.Sleep(1); // 시뮬레이션
    }
}

public class ProgressReport
{
    public long Current { get; set; }
    public long Total { get; set; }
    public double Percentage { get; set; }
    public string Message { get; set; }
}

4. 사용 예제 (콘솔 애플리케이션)​

class Program
{
    static async Task Main(string[] args)
    {
        Console.WriteLine("병렬 처리 시작 (아무 키나 누르면 취소)");
        
        using var cts = new CancellationTokenSource();
        
        // 키 입력 감지를 위한 백그라운드 태스크
        var keyTask = Task.Run(() =>
        {
            Console.ReadKey();
            cts.Cancel();
        });
        
        // 진행률 보고 핸들러
        var progress = new Progress<ProgressReport>(report =>
        {
            Console.WriteLine($"\r{report.Message} ({report.Percentage:F1}%)");
        });
        
        try
        {
            var processor = new AdvancedParallelProcessor();
            await processor.ProcessDataWithProgressAsync(
                itemCount: 100000,
                progress: progress,
                cancellationToken: cts.Token);
                
            Console.WriteLine("\n작업이 완료되었습니다!");
        }
        catch (OperationCanceledException)
        {
            Console.WriteLine("\n작업이 취소되었습니다.");
        }
        
        Console.WriteLine("아무 키나 누르면 종료합니다.");
        Console.ReadKey();
    }
}

3. 메모리 사용량 최적화​

// 대용량 데이터 처리 시 스트리밍 방식 사용
public async IAsyncEnumerable<TResult> ProcessLargeDataStreamAsync<TResult>(
    IAsyncEnumerable<TInput> source,
    [EnumeratorCancellation] CancellationToken cancellationToken = default)
{
    await foreach (var batch in source.Buffer(1000).WithCancellation(cancellationToken))
    {
        var results = new ConcurrentBag<TResult>();
        
        Parallel.ForEach(batch, new ParallelOptions 
        { 
            CancellationToken = cancellationToken 
        }, item =>
        {
            cancellationToken.ThrowIfCancellationRequested();
            var result = ProcessItem(item);
            results.Add(result);
        });
        
        foreach (var result in results)
        {
            yield return result;
        }
    }
}

🎯 결론​

핵심 요약​

현대적인 C# 개발에서는:

  1. Thread.Abort() 사용 금지: 안전하지 않고 .NET Core에서 지원되지 않음
  2. CancellationToken 활용: 안전하고 예측 가능한 작업 중단
  3. Task 기반 비동기 패턴: Thread보다 효율적이고 관리하기 쉬움
  4. 적절한 병렬도 설정: 성능과 리소스 사용량의 균형
  5. 진행률 보고: 사용자 경험 향상

이러한 패턴을 따르면 안전하고 효율적인 병렬 처리 애플리케이션을 개발할 수 있습니다.
