next-yak: Next.js를 위한 러스트 기반 Zero-runtime CSS-in-JS

2025년 5월 26일 · 약 13분
Jeongyong Park
쌍팔년생 개발자

CSS-in-JS는 현대 React 개발에서 널리 사용되지만, 런타임 성능 문제가 항상 고민거리였습니다. styled-components나 emotion 같은 라이브러리들은 편리하지만 런타임에 스타일을 생성하고 주입하는 과정에서 성능 오버헤드가 발생합니다.

next-yak는 이러한 문제를 해결하기 위해 탄생한 혁신적인 CSS-in-JS 라이브러리입니다. 러스트로 개발된 이 라이브러리는 빌드 타임에 CSS를 추출하여 런타임 JavaScript 오버헤드를 완전히 제거합니다.

이 글에서 다룰 내용:

  • next-yak의 Zero-runtime 아키텍처와 성능 이점
  • Next.js 프로젝트에서의 설치 및 설정 방법
  • React Server Components와의 완벽한 호환성
  • 실제 프로젝트에서 활용할 수 있는 고급 패턴들
  • 기존 CSS-in-JS 라이브러리에서의 마이그레이션 가이드

실제 수천 명의 사용자 데이터를 기반으로 한 성능 검증 결과:

  • LCP(Largest Contentful Paint): 20% 이상 향상
  • SSR 지연 시간: 15% 감소
  • INP(Interaction to Next Paint): 10% 이상 개선

설치 및 설정​

패키지 설치​

npm install next-yak
# 또는
yarn add next-yak
# 또는
pnpm add next-yak

Next.js 설정​

next.config.js 파일에 next-yak 플러그인을 추가합니다:

next.config.js
const { withYak } = require('next-yak/withYak');

/** @type {import('next').NextConfig} */
const nextConfig = {
  // 기존 설정들...
};

module.exports = withYak(nextConfig);

TypeScript 설정 (선택사항)​

TypeScript를 사용하는 경우 tsconfig.json에 타입 정의를 추가할 수 있습니다:

tsconfig.json
{
  "compilerOptions": {
    "types": ["next-yak"]
  }
}

주요 특징​

1. Zero-runtime 아키텍처​

기존 CSS-in-JS 라이브러리들과 달리 next-yak는 빌드 타임에 모든 스타일을 정적 CSS로 변환합니다:

// 개발 시 작성하는 코드
const Button = styled.button`
  background: blue;
  color: white;
`;

// 빌드 후 생성되는 코드
const Button = ({ children, ...props }) => (
  <button className="button-abc123" {...props}>
    {children}
  </button>
);

// 생성되는 CSS 파일
.button-abc123 {
  background: blue;
  color: white;
}

2. React Server Components 완벽 지원​

Next.js 13+의 App Router와 Server Components에서 완벽하게 작동합니다:

// app/components/ServerButton.tsx (Server Component)
import { styled } from 'next-yak';

const ServerButton = styled.button`
  background: linear-gradient(45deg, #fe6b8b 30%, #ff8e53 90%);
  border: 0;
  border-radius: 3px;
  color: white;
  height: 48px;
  padding: 0 30px;
`;

export default function Page() {
  return (
    <ServerButton>
      서버에서 렌더링되는 버튼
    </ServerButton>
  );
}

3. 동적 스타일링과 테마 지원​

props를 통한 동적 스타일링도 빌드 타임에 최적화됩니다:

import { styled, css } from 'next-yak';

const Card = styled.div<{
  $variant: 'primary' | 'secondary' | 'danger';
  $size: 'small' | 'medium' | 'large';
}>`
  padding: ${props => {
    switch (props.$size) {
      case 'small': return '8px 12px';
      case 'medium': return '12px 16px';
      case 'large': return '16px 24px';
      default: return '12px 16px';
    }
  }};
  
  border-radius: 8px;
  transition: all 0.2s ease;
  
  ${props => {
    switch (props.$variant) {
      case 'primary':
        return css`
          background: #007bff;
          color: white;
          &:hover { background: #0056b3; }
        `;
      case 'secondary':
        return css`
          background: #6c757d;
          color: white;
          &:hover { background: #545b62; }
        `;
      case 'danger':
        return css`
          background: #dc3545;
          color: white;
          &:hover { background: #c82333; }
        `;
      default:
        return css``;
    }
  }}
`;

실제 사용 예제​

1. 복잡한 컴포넌트 시스템​

import { styled, css } from 'next-yak';

// 기본 버튼 스타일
const BaseButton = styled.button`
  display: inline-flex;
  align-items: center;
  justify-content: center;
  border: none;
  border-radius: 6px;
  font-weight: 500;
  text-decoration: none;
  cursor: pointer;
  transition: all 0.2s ease;
  
  &:disabled {
    opacity: 0.6;
    cursor: not-allowed;
  }
`;

// 크기별 스타일
const sizeStyles = {
  small: css`
    padding: 6px 12px;
    font-size: 14px;
    min-height: 32px;
  `,
  medium: css`
    padding: 8px 16px;
    font-size: 16px;
    min-height: 40px;
  `,
  large: css`
    padding: 12px 24px;
    font-size: 18px;
    min-height: 48px;
  `,
};

// 변형별 스타일
const variantStyles = {
  primary: css`
    background: #007bff;
    color: white;
    &:hover:not(:disabled) {
      background: #0056b3;
    }
  `,
  outline: css`
    background: transparent;
    color: #007bff;
    border: 1px solid #007bff;
    &:hover:not(:disabled) {
      background: #007bff;
      color: white;
    }
  `,
  ghost: css`
    background: transparent;
    color: #007bff;
    &:hover:not(:disabled) {
      background: rgba(0, 123, 255, 0.1);
    }
  `,
};

// 최종 버튼 컴포넌트
const Button = styled(BaseButton)<{
  $size?: keyof typeof sizeStyles;
  $variant?: keyof typeof variantStyles;
}>`
  ${props => sizeStyles[props.$size || 'medium']}
  ${props => variantStyles[props.$variant || 'primary']}
`;

// 사용 예시
export default function ButtonExample() {
  return (
    <div style={{ display: 'flex', gap: '12px', flexWrap: 'wrap' }}>
      <Button $size="small" $variant="primary">
        작은 버튼
      </Button>
      <Button $size="medium" $variant="outline">
        중간 버튼
      </Button>
      <Button $size="large" $variant="ghost">
        큰 버튼
      </Button>
    </div>
  );
}

성능 최적화 전략​

1. 빌드 타임 최적화​

next-yak는 빌드 과정에서 다음과 같은 최적화를 수행합니다:

// next.config.js에서 최적화 옵션 설정
const { withYak } = require('next-yak/withYak');

const nextConfig = {
  experimental: {
    // CSS 최적화 활성화
    optimizeCss: true,
  },
};

module.exports = withYak(nextConfig, {
  // next-yak 특정 옵션들
  cssOptimization: {
    // 중복 스타일 제거
    removeDuplicates: true,
    // CSS 최소화
    minify: true,
    // 사용하지 않는 스타일 제거
    purgeUnused: true,
  },
});

2. 코드 분할과 지연 로딩​

import { styled } from 'next-yak';
import { lazy, Suspense } from 'react';

// 지연 로딩되는 컴포넌트

}

기존 라이브러리와의 비교​

styled-components vs next-yak​

특징styled-componentsnext-yak
런타임 성능런타임에 CSS 생성빌드 타임 CSS 추출
번들 크기~40KB~2KB
SSR 지원복잡한 설정 필요기본 지원
RSC 지원제한적완벽 지원
개발자 경험우수동일한 API

마이그레이션 가이드​

기존 styled-components 프로젝트에서 next-yak로 마이그레이션하는 방법:

// Before (styled-components)
import styled from 'styled-components';

const Button = styled.button`
  background: ${props => props.primary ? 'blue' : 'white'};
  color: ${props => props.primary ? 'white' : 'blue'};
`;

// After (next-yak)
import { styled } from 'next-yak';

const Button = styled.button<{ $primary?: boolean }>`
  background: ${props => props.$primary ? 'blue' : 'white'};
  color: ${props => props.$primary ? 'white' : 'blue'};
`;

// 사용법 변경
// Before: <Button primary>Click me</Button>
// After: <Button $primary>Click me</Button>

고급 사용법​

1. 테마 시스템 구축​

// theme.ts
export const theme = {
  colors: {
    primary: '#007bff',
    secondary: '#6c757d',
    success: '#28a745',
    danger: '#dc3545',
    warning: '#ffc107',
    info: '#17a2b8',
  },
  spacing: {
    xs: '4px',
    sm: '8px',
    md: '16px',
    lg: '24px',
    xl: '32px',
  },
  breakpoints: {
    sm: '576px',
    md: '768px',
    lg: '992px',
    xl: '1200px',
  },
} as const;

// ThemeProvider.tsx
import { createContext, useContext } from 'react';

const ThemeContext = createContext(theme);

export const useTheme = () => useContext(ThemeContext);

export const ThemeProvider = ({ children }: { children: React.ReactNode }) => (
  <ThemeContext.Provider value={theme}>
    {children}
  </ThemeContext.Provider>
);

// 테마를 사용하는 컴포넌트
import { styled } from 'next-yak';
import { useTheme } from './ThemeProvider';

const ThemedButton = styled.button<{ $variant: keyof typeof theme.colors }>`
  padding: ${() => {
    const theme = useTheme();
    return `${theme.spacing.sm} ${theme.spacing.md}`;
  }};
  
  background: ${(props) => {
    const theme = useTheme();
    return theme.colors[props.$variant];
  }};
  
  @media (min-width: ${() => {
    const theme = useTheme();
    return theme.breakpoints.md;
  }}) {
    padding: ${() => {
      const theme = useTheme();
      return `${theme.spacing.md} ${theme.spacing.lg}`;
    }};
  }
`;

const Button = styled.button<{ $color?: string }>`

결론​

next-yak는 Next.js 프로젝트에서 CSS-in-JS의 성능 문제를 근본적으로 해결하는 혁신적인 솔루션입니다. 러스트 기반의 고성능 아키텍처와 zero-runtime 접근 방식을 통해:

  • 개발자 경험: styled-components와 동일한 친숙한 API
  • 성능 최적화: 런타임 오버헤드 완전 제거
  • 미래 지향성: React Server Components 완벽 지원
  • 확장성: 대규모 프로젝트에서도 안정적인 성능

특히 성능이 중요한 프로덕션 환경에서 기존 CSS-in-JS 라이브러리들의 한계를 뛰어넘는 탁월한 대안이 될 수 있습니다. 점진적 마이그레이션이 가능하므로 기존 프로젝트에서도 부담 없이 도입할 수 있습니다.
